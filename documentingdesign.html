Documenting Design

“If it isn’t written down, it didn’t happen.”

This seminar taught us about the importance to document our design process and exposed us to tools to do so.

Documentation transforms chaos into clarity. It provides a clear and comprehensive understanding of the design process, from ideation, to research, prototypes and interventions.

Documentation transforms design decisions into a shared language. Documentation serves as a communication tool between designers and and other stakeholders. It ensures that everyone involved in the project understands the goals, processes, requirements, and specifications, fostering collaboration and preventing misunderstandings. It creates open knowledge.

Documenting is a love letter that you write to your future self. Although this process is eating up a lot of storage space on my phone, I'm really grasping the value of thorough documentation as a way to understand, remember and analyze my design process for better end results.

Documenting tools

We were introduced to programming to build our website to document our MDEF journey. We used github to create a domain for our website and cloned a website template through our computer directory. We then used Visual Studio Code and MkDocs to populate our website. However, I decided to use Mobirise, a website building app, to build this website as the outcome is more visual and the user interface is easier to use.
